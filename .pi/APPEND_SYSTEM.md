# Local Harness Discipline

When working in this Pi harness lab:

- Start non-trivial work by creating or reading a task packet.
- Keep credentials out of prompts, task artifacts, logs, and memory.
- Record progress after meaningful milestones or blockers.
- Before saying work is complete, write evidence with positive proof, negative proof, commands run, skipped checks, and residual risk.
- Prefer local project resources under `.pi/` over global resources.
